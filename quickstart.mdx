## Getting Started

This guide will walk you through using the Pseudorandom Number Generation Simulator to generate your first sequence of pseudorandom numbers.

<Steps>
  <Step title="Open the Application">
    Open the `Index.html` file in your web browser. You'll see a form with several input fields and an algorithm selector.
  </Step>

  <Step title="Select an Algorithm">
    Choose between two available algorithms:
    
    - **Cuadrados Medios** - Uses a single seed value
    - **Productos Medios** - Requires two seed values
    
    <Note>
    When you select "Productos Medios", a second seed input field will automatically appear.
    </Note>
  </Step>

  <Step title="Configure Parameters">
    Fill in the required parameters for your simulation:
    
    - **Cantidad de números a generar** - How many random numbers you want to generate
    - **Semilla 1** - First seed value (required for both algorithms)
    - **Semilla 2** - Second seed value (only for Productos Medios)
    - **Valor de D** - Number of digits to extract from the middle
  </Step>

  <Step title="Generate Results">
    Click the **Generar** button to run the simulation and view results in a table format.
  </Step>
</Steps>

## Example: Middle Squares Method

Let's generate 5 pseudorandom numbers using the Cuadrados Medios algorithm.

### Input Parameters

```text
Algorithm: Cuadrados Medios
Cantidad de números: 5
Semilla 1: 1234
Valor de D: 4
```

### How It Works

The algorithm executes the following logic from `index.js:48-74`:

```javascript
function simularCuadradosMedios(cantidadNumeros, semilla, d) {
    var semillaActual = semilla;
    for (var i = 0; i < cantidadNumeros; ++i) {
        var yi = semillaActual * semillaActual;  // Square the seed
        var yiStr = yi.toString();
        var inicio = (yiStr.length - d) / 2;     // Find middle position
        var xi = parseInt(yiStr.substr(inicio, d)); // Extract D digits
        var ri = xi / Math.pow(10, d);           // Normalize to [0,1)
        
        semillaActual = xi;  // Use xi as next seed
    }
}
```

### Understanding the Output

The results table shows three columns:

| Column | Description | Example |
|--------|-------------|---------|
| **yi** | The squared value (semilla × semilla) | 1522756 |
| **xi** | The middle D digits extracted from yi | 5227 |
| **ri** | The normalized random number (xi / 10^D) | 0.5227 |

<Info>
Each iteration uses the `xi` value from the previous iteration as the new seed, creating a deterministic sequence.
</Info>

## Example: Middle Products Method

Now let's try the Productos Medios algorithm with two seeds.

### Input Parameters

```text
Algorithm: Productos Medios
Cantidad de números: 5
Semilla 1: 1234
Semilla 2: 5678
Valor de D: 4
```

### How It Works

The algorithm from `index.js:76-105`:

```javascript
function simularProductosMedios(cantidadNumeros, semilla1, semilla2, d) {
    var semillaActual1 = semilla1;
    var semillaActual2 = semilla2;

    for (var i = 0; i < cantidadNumeros; ++i) {
        var yi = semillaActual1 * semillaActual2;  // Multiply seeds
        var yiStr = yi.toString();
        var inicio = (yiStr.length - d) / 2;       // Find middle
        var xi = parseInt(yiStr.substr(inicio, d)); // Extract D digits
        var ri = xi / Math.pow(10, d);             // Normalize
        
        semillaActual1 = semillaActual2;  // Rotate: seed2 becomes seed1
        semillaActual2 = xi;               // xi becomes new seed2
    }
}
```

### Key Difference

<CodeGroup>

```javascript Cuadrados Medios
var yi = semillaActual * semillaActual;  // Square single seed
semillaActual = xi;                       // Simple replacement
```

```javascript Productos Medios
var yi = semillaActual1 * semillaActual2; // Multiply two seeds
semillaActual1 = semillaActual2;          // Seed rotation
semillaActual2 = xi;                       // Update second seed
```

</CodeGroup>

The Productos Medios method uses a **rotation scheme** that can help avoid some degeneracy issues present in the simple Middle Squares method.

## Tips for Best Results

<AccordionGroup>
  <Accordion title="Choosing Good Seed Values">
    - Avoid seeds with many zeros (e.g., 1000, 0100)
    - Use seeds that are relatively prime to each other for Productos Medios
    - Ensure seeds have the same number of digits as your D parameter
  </Accordion>

  <Accordion title="Selecting the D Parameter">
    - D represents the precision of your random numbers
    - Common values: 4, 6, or 8 digits
    - Larger D values require proportionally larger seeds
    - D should be even for symmetric extraction from middle
  </Accordion>

  <Accordion title="Understanding Period Length">
    - These algorithms have finite periods before repeating
    - Maximum possible period is 10^D
    - Most sequences have much shorter periods
    - Monitor for degeneracy (sequence reaches 0)
  </Accordion>
</AccordionGroup>

## Common Issues

<Warning>
**Degeneracy**: If your sequence produces all zeros, the algorithm has degenerated. This happens when:
- The seed is too small relative to D
- The extracted middle digits become 0000...
- Solution: Choose a different seed value
</Warning>

<Note>
**Short Periods**: If you notice the sequence repeating quickly, try:
- Using different seed values
- Switching to the Productos Medios algorithm
- Adjusting the D parameter
</Note>

## Understanding the Results Table

The application displays results dynamically using `index.js:36`:

```javascript
var resultadosDiv = document.getElementById('tablaResultados');
```

Each row in the results table represents one iteration:

- **yi**: Shows the intermediate calculation (square or product)
- **xi**: The extracted middle digits, formatted with leading zeros using `padStart(d, '0')`
- **ri**: The final pseudorandom number, formatted to D decimal places using `toFixed(d)`
